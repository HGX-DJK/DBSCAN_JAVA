# 高性能空间聚类分析平台 (Urban Spatial Clustering)

这是一个基于 Java Spring Boot 与 Maptalks 的高性能城市级空间聚类分析工具。系统从零实现了工业级的 **DBSCAN** 和 **HDBSCAN** 聚类算法，专为处理数十万级的大规模城市地理点云数据（如案件频发地、交通热点、人口分布等）设计。

## 🌟 核心特性

### 1. 极致优化的底层算法引擎
* **完全重构的 HDBSCAN 算法**：
  * **压缩层次树 (Condensed Tree) 提取**：摒弃了简单的树搜索，完全实现了标准的自底向上“稳定性质量过剩 (Mass Excess)”积分机制。能够精确分辨并在大聚类中提取出局部高密度微聚类，彻底消灭“大一统”的聚类死结。
  * **MST 防御性逻辑**：修复了最小生成树(MST)层级构建中可能出现的密度反转问题。
* **O(log N) 的极速 KD-Tree**：
  * **Quickselect 建树**：使用 `O(n)` 的快速选择算法寻找中位数划分，将建树时间从 `O(n log^2 n)` 降至 `O(n log n)`。
  * **投影坐标系剪枝修复**：修复了在经纬度体系下搜索半径的单位错位Bug。算法全局基于公里坐标系(KM)运行，搜索剪枝效率呈现指数级增长。

### 2. 城市级地理空间投影 (Urban Projection)
地球是球体，直接在经纬度上使用欧氏距离会导致高纬度区域的严重畸变。
本系统会在聚类前，自动提取整个数据集的**平均纬度**，以此为基准采用墨卡托思想将所有经纬度无损映射为**以千米(KM)为单位的二维平面坐标系**。
* **效果**：无论数据在赤道还是高纬度，设定的 `epsilon` (例如 0.5 千米) 代表的物理距离将永远绝对精确。

### 3. 后处理与洗牌机制
算法在将结果返回前端之前，包含了一个强健的 `ClusterService` 拦截过滤层：
* **二次噪声过滤**：强制剔除由于算法波动残留的、不满足 `minClusterSize` 阈值的极小碎簇。
* **连续性 ID 重映射**：将打散的集群 ID 重新洗牌，确保前端图例永远渲染为连续的 `0, 1, 2...`，大幅提升数据可视化面板的整洁度。

### 4. 现代化的 WebGL 前端可视化
* **Maptalks 地图引擎**：舍弃了传统的 DOM 标记图层 (Leaflet)，采用基于 WebGL 的 Maptalks 矢量图层批量渲染引擎，支持单页数十万标记点的丝滑缩放与 3D 视角拖拽。
* **高交互 UI**：
  * 采用现代 **毛玻璃 (Glassmorphism)** 效果的悬浮图例组件。
  * 自适应弹性防溢出的图例滚动条。
  * 精准的交互式数据气泡弹窗。

## 🚀 快速开始

### 运行环境
* **JDK**: Java 17+
* **构建工具**: Maven 3.6+
* **浏览器**: 推荐使用最新版的 Chrome / Edge

### 启动项目
1. 在项目根目录执行 Maven 编译：
   ```bash
   mvn clean package
   ```
2. 运行 Spring Boot 应用：
   ```bash
   java -jar target/dbscan-1.0.jar
   ```
3. 打开浏览器访问：`http://localhost:8080/`

### 使用流程
1. 在页面左侧点击 **[选择 CSV 文件]**，上传包含地理位置的数据集。
2. 确认输入框中的**经度字段名**和**纬度字段名**与 CSV 表头一致（默认 `lng` 和 `lat`）。
3. 选择您想要的算法：
   * **DBSCAN**：适合固定空间半径（如：寻找彼此相距不超过 1 千米的事件组）。需要设置 `epsilon` 距离阈值。
   * **HDBSCAN**：适合密度极不均匀、存在多个不同层级热点的城市数据（系统主推算法）。无需设置 `epsilon`，只需设定您认为“多少个点能构成一个有意义的热点 (`minClusterSize`)”。
4. 点击 **[执行聚类分析]**，等待引擎极速计算，并在右侧地图上查看带图例分类的渲染结果。

## 📝 参数解析调优建议
| 参数名 | 说明 | 调优建议 |
| :--- | :--- | :--- |
| **距离阈值 (eps)** | *仅用于 DBSCAN*。两个点被视为“邻居”的最大绝对地理距离。单位：千米。 | 城市街区分析建议设为 `0.5` ~ `2.0`。 |
| **最小聚类大小** | *HDBSCAN 核心参数*。一个群体要被称为“有效集群”，至少需要包含的成员数量。 | 越大则过滤越严格，展示的均为大热点，建议 `5` ~ `20`。 |
| **最小样本数 (minPts)** | 用于计算初始密度的 K-NN 邻居数。决定了算法对局部噪声的敏感度。 | 默认等于最小聚类大小。如果数据量巨大且极度密集，可适当调小以寻找更精细的热点。 |
